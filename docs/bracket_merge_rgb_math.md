# 包围曝光合成算法和参数计算说明

本文按当前项目实现解释包围曝光合成。核心问题可以简化成：

同一个像素位置 `(x, y)`，有 5 张 JPG。每张都有一个 RGB 值：

```text
P1 = (R1, G1, B1)
P2 = (R2, G2, B2)
P3 = (R3, G3, B3)
P4 = (R4, G4, B4)
P5 = (R5, G5, B5)
```

合成算法要算出最终：

```text
Pout = (Rout, Gout, Bout)
```

在代码里，这个过程主要在 `photo_share/bracket_merge.py`。

## 总流程

当前合成顺序是：

```text
读取 JPG -> EXIF 旋转校正 -> 尺寸统一 -> 对齐 -> 算法合成 -> 后期参数调整 -> 保存 JPG
```

更具体：

1. 读取每张 JPG，得到 0 到 255 的 RGB 数组。
2. 如果尺寸不同，按第一张图尺寸缩放统一。
3. 根据“对齐”选项，把图片尽量挪到同一个坐标系。
4. 根据“算法”选项得到一张初始合成图。
5. 对初始合成图应用曝光、阴影、高光、对比度、饱和度、锐化。
6. 保存为 JPG，质量由“质量”参数控制。

## 对齐选项

对齐发生在合成算法之前。它不是直接改 RGB 亮度，而是改变“第 i 张图的哪个像素对应当前输出像素”。

### 关闭

不做对齐。

```text
第 i 张图用于合成的像素 = 第 i 张图原始位置 (x, y)
```

优点是不会把画面扭坏。缺点是手持拍摄容易重影。

### 简单

以中间那张为参考图。5 张图时，参考图通常是第 3 张。

流程：

1. 用 ORB 找特征点。
2. 匹配每张图和参考图的特征点。
3. 优先估计单应性变换。
4. 如果失败，估计仿射变换。
5. 再失败，用相位相关估计平移。

最终效果是：背景、山、建筑这种刚体区域通常能对齐。

### 人物/轻微移动

先执行“简单”对齐，然后在 Fusion 合成时启用去重影权重。

它不会强行把人物手脚、树叶、路人局部拉伸对齐。它做的是：

```text
如果某个局部区域和参考帧差异很大，
说明这里可能有人、树叶、反光或局部运动，
则降低其他帧在这个区域的权重，
让参考帧占主导。
```

所以它适合轻微人物移动、树叶动、手持轻微位移。它不是“局部肢体变形矫正”，而是“局部去重影”。

### 复杂

使用 SIFT/AKAZE/ORB、多轮 RANSAC/MAGSAC、ECC、光流细化。

它可能对纯背景更强，但对人物、玻璃、前景遮挡、视差大的场景容易过度扭曲。复杂不是默认推荐。

## Fusion 算法

Fusion 不是先恢复真实 HDR 辐射值，而是直接在 JPG 像素上做加权融合。

当前普通 Fusion 使用 OpenCV `MergeMertens`：

```python
cv2.createMergeMertens(
    contrast_weight=1.0,
    saturation_weight=0.15,
    exposure_weight=1.2,
)
```

对单个像素，可以理解为：

```text
Pout = W1 * P1 + W2 * P2 + W3 * P3 + W4 * P4 + W5 * P5
```

其中：

```text
W1 + W2 + W3 + W4 + W5 = 1
```

每个权重 `Wi` 由三个因素决定：

```text
Wi = contrast_score ^ 1.0
   * saturation_score ^ 0.15
   * exposure_score ^ 1.2
```

直觉解释：

- `contrast_score`：局部细节越明显，权重越高。
- `saturation_score`：颜色越有区分度，权重略高。
- `exposure_score`：亮度越接近中间灰，权重越高。

比如某个像素在 5 张图里的亮度大概是：

```text
P1 = 很暗，树根有细节但接近黑
P2 = 偏暗，树根细节较好
P3 = 正常，草地正常
P4 = 偏亮，雪山清楚但草地可能过亮
P5 = 很亮，天空和雪山可能接近白
```

Fusion 会倾向选择：

- 雪山区域：从较暗或正常曝光的图里取，因为亮图可能过曝。
- 树根区域：从较亮图里取，因为暗图接近黑。
- 草地中间调：从正常曝光图里取。

但 Fusion 不保证一定把暗部拉得很亮。它只是挑“权重合适”的像素混合。如果树根在所有图里都偏暗，或者局部权重仍偏向中间曝光，最后树根仍然会黑。这就是为什么还需要“阴影”参数。

## 人物/轻微移动 Fusion

这个模式是项目里的自定义 Fusion 变体。

先计算每张图的亮度：

```text
Li = 0.2126 * Ri + 0.7152 * Gi + 0.0722 * Bi
```

然后计算两个基础分数。

局部对比度：

```text
contrast_i = abs(Li - blur(Li))
```

曝光适中分数：

```text
exposure_i = exp(-((Li - 128)^2) / (2 * 72^2))
```

亮度越接近 128，`exposure_i` 越高。

基础权重：

```text
Wi = contrast_i * 0.7 + exposure_i * 0.3 + 0.04
```

然后做运动检测。以中间参考帧为例：

```text
diff_i = abs(Li - L_reference)
local_diff_i = blur(diff_i)
```

如果：

```text
local_diff_i > 20
```

就认为该局部可能有运动或重影风险。非参考帧会被惩罚：

```text
penalty_i = exp(-((local_diff_i / 28)^2))
Wi = Wi * penalty_i
```

如果某个位置任意一张图都被判断有运动，这个区域会膨胀一圈，然后：

```text
所有帧权重 *= 0.25
参考帧权重至少设为 1.0
```

最后归一化：

```text
Wi = Wi / (W1 + W2 + W3 + W4 + W5)
```

输出：

```text
Rout = W1*R1 + W2*R2 + W3*R3 + W4*R4 + W5*R5
Gout = W1*G1 + W2*G2 + W3*G3 + W4*G4 + W5*G5
Bout = W1*B1 + W2*B2 + W3*B3 + W4*B4 + W5*B5
```

所以这个模式本质上仍然是加权平均，只是运动区域会更相信参考帧。

## Debevec HDR

Debevec 是更传统的 HDR 方法。它不是直接平均 JPG RGB，而是尝试估计“真实场景辐射值”。

对某个颜色通道，比如红色 R：

```text
Z1 = R1
Z2 = R2
Z3 = R3
Z4 = R4
Z5 = R5
```

同时需要每张图的曝光时间：

```text
t1, t2, t3, t4, t5
```

Debevec 会先估计相机响应曲线：

```text
Z = f(E * t)
```

其中：

- `Z` 是 JPG 像素值，0 到 255。
- `E` 是场景真实辐射值。
- `t` 是曝光时间。
- `f` 是相机/JPG 的响应曲线。

估计出响应曲线后，反推每张图对应的辐射值：

```text
E_i = inverse_f(Z_i) / t_i
```

再把 5 个 `E_i` 合并成 HDR 值：

```text
E_hdr = weighted_average(E1, E2, E3, E4, E5)
```

通常中间亮度像素权重更高，接近 0 或 255 的像素权重更低，因为它们可能欠曝或过曝。

Debevec 得到的是 HDR 浮点图，不是 0 到 255 的普通 JPG，所以还要色调映射。当前代码使用：

```python
cv2.createTonemapReinhard(
    gamma=1.0,
    intensity=0.0,
    light_adapt=0.8,
    color_adapt=0.0,
)
```

色调映射会把 HDR 压回普通显示范围：

```text
HDR float RGB -> 0 到 255 RGB
```

Debevec 的优点是理论更完整。缺点是 JPG 已经被机内处理过，曝光时间不准或曲线估计不好时，结果可能偏灰、偏色或局部不自然。

## Robertson HDR

Robertson 和 Debevec 目标类似，也是恢复 HDR 辐射值：

```text
JPG 像素 + 曝光时间 -> 场景 HDR 辐射值 -> 色调映射 -> JPG
```

区别是 Robertson 用迭代方式同时估计响应曲线和辐射值。它通常更稳一些，但可能更慢。

单像素理解仍然是：

```text
Z1..Z5 + t1..t5
-> 估计响应曲线
-> 反推出 E1..E5
-> 合成 E_hdr
-> 色调映射成 RGB
```

如果 EXIF 里没有可靠曝光时间，当前代码会生成一组备用曝光时间：

```text
5 张图时大致按：1/4, 1/2, 1, 2, 4
```

这只是兜底假设，不一定和真实拍摄一致。因此 Debevec/Robertson 在 JPG 上不一定比 Fusion 更好。

## 后期参数

不管前面用 Fusion、Debevec 还是 Robertson，得到初始 RGB 图后，都会进入同一套后期参数。

当前顺序：

```text
曝光 -> 阴影/高光 -> 对比度 -> 饱和度 -> 锐化 -> JPG质量保存
```

### 曝光

参数范围：`-2.0 到 2.0`

计算近似是：

```text
RGB = RGB * (2 ^ exposure)
```

例子：

```text
exposure = 1.0  -> RGB * 2
exposure = -1.0 -> RGB * 0.5
exposure = 0    -> 不变
```

这是整体变亮或变暗，不区分天空和树根。

### 阴影

参数范围：`0 到 1`

先计算亮度：

```text
L = 0.2126*R + 0.7152*G + 0.0722*B
N = L / 255
```

阴影遮罩：

```text
shadow_mask = clamp((0.62 - N) / 0.62, 0, 1) ^ 1.6
```

越暗的像素，`shadow_mask` 越接近 1。亮部接近 0。

提亮倍率：

```text
lift = 1 + shadows * 1.15 * shadow_mask
RGB = RGB * lift
```

例子：如果某个树根像素很暗：

```text
RGB = (20, 35, 18)
shadow_mask 接近 1
shadows = 0.5
lift = 1 + 0.5 * 1.15 = 1.575
新 RGB ≈ (31, 55, 28)
```

如果是天空亮部，`shadow_mask` 接近 0，基本不变。

### 高光

参数范围：`0 到 1`

高光遮罩：

```text
highlight_mask = clamp((N - 0.58) / 0.42, 0, 1) ^ 1.4
```

越亮的像素，`highlight_mask` 越接近 1。

压高光：

```text
compression = 1 - highlights * 0.55 * highlight_mask
RGB = RGB * compression + 255 * highlights * 0.06 * highlight_mask
```

直觉是：很亮的地方会被压一点，避免天空和雪山继续冲白。

### 对比度

参数范围：`0.5 到 2.0`

Pillow 的对比度增强可理解为围绕中间灰拉伸：

```text
RGB = 128 + contrast * (RGB - 128)
```

例子：

```text
contrast = 1.0 -> 不变
contrast = 1.2 -> 暗部更暗，亮部更亮
contrast = 0.8 -> 暗部和亮部都向中间靠
```

如果你已经把阴影拉起来，但画面发灰，可以稍微加对比度，比如 `1.05` 到 `1.15`。

### 饱和度

参数范围：`0 到 2.0`

Pillow 的饱和度增强可理解为：

```text
灰度 = RGB 转灰
RGB = gray + saturation * (RGB - gray)
```

例子：

```text
saturation = 1.0 -> 不变
saturation = 1.2 -> 颜色更鲜
saturation = 0.8 -> 颜色更淡
saturation = 0   -> 黑白
```

风景图如果草地太假，降低饱和度。如果画面偏灰，适当增加。

### 锐化

参数范围：`0 到 2.0`

当前调用：

```text
Pillow Sharpness factor = 1 + sharpen
```

所以：

```text
sharpen = 0   -> factor 1.0，基本不锐化
sharpen = 0.2 -> factor 1.2，轻微锐化
sharpen = 1.0 -> factor 2.0，较强锐化
```

锐化不会恢复过曝或欠曝细节，只会强化边缘。过高会让树叶、草地出现硬边或噪点。

### 质量

参数范围：`70 到 98`

这是最终 JPG 保存质量。它不参与合成，只影响输出文件压缩。

```text
quality 越高 -> 文件越大，压缩损失越少
quality 越低 -> 文件越小，压缩痕迹越明显
```

建议普通预览用 `92`。需要更高质量可用 `95` 到 `98`。

## 用哪个算法

### 风景、大光比、JPG 包围曝光

优先：

```text
Fusion + 简单对齐
```

或有树叶/人物轻微移动：

```text
Fusion + 人物/轻微移动
```

如果暗部太黑：

```text
阴影 0.4 到 0.7
高光 0.15 到 0.35
对比度 1.0 到 1.1
```

### 想要更 HDR 的效果

可以试：

```text
Robertson
Debevec
```

但 JPG 输入下，它们不一定更好。它们更依赖曝光时间和响应曲线估计。

### 有明显人物移动

优先：

```text
Fusion + 人物/轻微移动
```

避免：

```text
复杂对齐
```

复杂对齐可能把人物、树叶、玻璃反射扭曲。

## 一个具体像素例子

假设某个树根像素在 5 张图中的 RGB 是：

```text
P1 = (5, 12, 6)
P2 = (12, 25, 14)
P3 = (25, 48, 28)
P4 = (55, 88, 50)
P5 = (90, 130, 80)
```

Fusion 会给每张一个权重，例如：

```text
W = (0.02, 0.08, 0.22, 0.38, 0.30)
```

那么：

```text
Rout = 5*0.02 + 12*0.08 + 25*0.22 + 55*0.38 + 90*0.30
     = 54.46

Gout = 12*0.02 + 25*0.08 + 48*0.22 + 88*0.38 + 130*0.30
     = 85.24

Bout = 6*0.02 + 14*0.08 + 28*0.22 + 50*0.38 + 80*0.30
     = 50.40
```

初始合成：

```text
Pout ≈ (54, 85, 50)
```

如果 `阴影 = 0.5`，这个像素亮度偏暗，可能得到：

```text
lift ≈ 1.35
Pout ≈ (73, 115, 68)
```

这就是为什么你截图里“雪山清楚但树根太黑”时，应该优先调“阴影”，而不是只加整体曝光。整体曝光会把雪山和天空也一起推亮，容易过曝。

## 当前推荐默认理解

把三种算法理解成：

```text
Fusion:
  直接从 5 张 JPG 的 RGB 里按权重挑好看的部分混合。

Debevec:
  先估计相机响应曲线，反推出 HDR 辐射值，再压回 JPG。

Robertson:
  类似 Debevec，但用迭代法估计响应和 HDR，通常更慢。
```

把参数理解成：

```text
曝光:
  整张图一起变亮/变暗。

阴影:
  只重点提亮暗部，比如树根、树林。

高光:
  重点压住亮部，比如天空、雪山。

对比度:
  拉开或压缩明暗差。

饱和度:
  调颜色浓淡。

锐化:
  强化边缘，不恢复曝光细节。

质量:
  控制 JPG 保存压缩，不改变合成逻辑。
```
